# puma_cbsa_crosswalk -- data dictionary

One row per PUMA. Delivered as `puma_cbsa_crosswalk.parquet` (zstd) and
`puma_cbsa_crosswalk.csv` (identical content; CSV writes empty string for NA).

Generated by `build.R` from the three bundled inputs in `../source/`. This build
reproduces the crosswalk used in the AEI Housing Center's NLIHC Gap Report 2026
replication **exactly** (verified row-for-row against the parent pipeline's
`puma_cbsa_crosswalk.parquet`; 0 differing cells across all columns).

## Columns

| Column | Type | Description |
| ------ | ---- | ----------- |
| `puma_id` | chr(7) | Join key: `statefip` (2) + `puma` (5), zero-padded. Unique per row. |
| `statefip` | chr(2) | State FIPS code, zero-padded (includes DC = `11`, PR = `72`). |
| `puma` | chr(5) | PUMA code (2022 vintage), zero-padded. |
| `puma_name` | chr | PUMA name from the source crosswalk. `NA` for a PUMA carried in without a name (1 row). |
| `cbsa` | chr(5) or NA | Assigned CBSA code (2020 vintage). `NA` when the PUMA is non-metropolitan. |
| `cbsa_name` | chr | CBSA title, or the literal `"Non-metropolitan"` when `cbsa` is `NA`. |
| `overlap_pct` | dbl or NA | Population share of the PUMA in its assigned CBSA (the winning allocation factor), in [0.50, 1.00]. `NA` for non-metro PUMAs. |
| `is_metro` | lgl | `TRUE` when the PUMA is assigned to any CBSA (metropolitan OR micropolitan); equals `!is.na(cbsa)`. |
| `is_micro` | lgl | `TRUE` only for micropolitan CBSAs (OMB type M2). See the `NA` note below. |

**True metropolitan** PUMAs are `is_metro & !is_micro`. `is_metro` alone also
includes micropolitan areas.

## Note on `is_micro = NA` (35 PUMAs, 16 CBSAs)

Geocorr assigns CBSA codes from the **2020** OMB delineation (`cbsa20`), while the
metro/micro type is read from the **2023** delineation. Sixteen CBSAs present in
the 2020 vintage are absent from (renamed/renumbered in) the 2023 file, so their
metro/micro type is unknown and `is_micro` is `NA` (they remain `is_metro = TRUE`
via their CBSA assignment). This is a faithful reproduction of the parent
pipeline's behavior, which had the same `NA` values from its `tigris` 2023 lookup
-- it is not introduced by the offline delineation approach. Treat `is_micro = NA`
as "metropolitan status assigned, micro/metro split unavailable."
